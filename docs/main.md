# ZC

Проект ZC - диалект языка Си, который нацелен на улучшение лаконичности внутренних API и предоставления удобной стандартной библиотеки (STL), а также систему модулей. В STL на данный момент пристутсвуют модули:

- async
    
    Модуль `async` предоставляет базовые операции с асинхронным программированием, такие как **создание асинхронных функций, корутин с прерываниями (suspend)** и **ожидание выполнения корутин**. Также есть подмодуль `async_ring`, который нацелен на интеграцию `async` системы с `epoll`

- containers

    Модуль `containers` предоставляет набор разных базовых контейнеров, такие как: 
    - `array` (generic список, может хранить данные любого типа)
    - `kvtable` (таблица с парами ключ-значение, доступ за O(n))
    - `queue` (простая FIFO очередь)
    - `hashtable` (хештаблица с доступом за O(1) и O(n) в худшем случае)

- crpyto

    Модуль `crpyto` предоставляет пока что только хеш-функции:
    - `fletcher32` и `fletcher32_string` - функции для хеширования данных по алгоритму `fletcher` 

- debug.testing

    Модуль `debug.testing` предоставляет базовую тестовую систему, позволяя удобно создавать юнит тесты

- filesystem

    Модуль `filesystem` предоставляет базовые операции с файловой системой, такие как:
    - создание папок
    - редактирование, чтение файлов
    - перебор файлов и под-директорий в директории (как рекурсивно так и нет)
    - проверка пути на существование

- io

    Модуль `io` предоставляет несколько подмодулей:
    - `term`: содержит функции для взаимодействия с stdin и stdout
    - `sio`: содержит функции для как блокирующего так и неблокирующего ввода/вывода через потоки (объекты, хранящие файловые дескрипторы)
    - `epoller`: содержит функции для использования системы epoll

- math

    Модуль `math` предоставляет пока что только функции ГПСЧ (генератора псведо-случайных чисел): *tinymt32*, *micro_rand*, *mulberry32*, *mwc64x*

- memory

    Модуль `memory` предоставляет несколько подмодулей:
    - `allocators`: содержит несколько аллокаторов: 
      - GPA (General Purpose Allocator, указатели хранятся в хештаблице)
      - Арена-аллокатор
      - Basic-аллокатор (более простая и иногда более стабильная версия GPA)
      - Raw-аллокатор (является оберткой над стандартными функциями libc)
    - `sentinel`: помогает проверять данные на содержание заканчивающего элемента (сентинеля), а также добавлять его в новые данные
    - `slices`: помогает разделять данные на куски с заданными границами
    - `tokenizer`: помогает создавать slices, разбивая входные данные по токенам

- net

    Модуль `net` предоставляет базовые сетевые взаимодействия (клиент/сервер через TCP/IP)

- string

    Модуль `string` предоставляет тип 2 подмодуля:
    - **str**: тип `String`, а также базовые функции для работы с ним, например:
      - является ли строка числом, целым числом, числом с плавующей точкой
      - найти индекс первого вхождения элемента в строке
      - посчитать количество вхождений элемента в строке
      - ... 
    - **fmt**: предоставляет удобную функцию для форматирования строк по шаблонам

- system

    Модуль `system` предоставляет подмодуль `signals`, который помогает обрабатывать системные прервыания

- threading

    Модуль `threading` предоставляет несколько подмодулей:
    - **conditions**: условная переменная
    - **mutexes**: мьютексы для защиты данных в многопоточном коде
    - **awaiter**: обертка над `cond_t` и `mutex_t`, позволяющий блокировать выполнение кода до тех пор, пока заданная условная переменная не будет установлена
    - **mdthreads**: удобное и лаконичное API для POSIX потоков

- time

    Модуль `time` предоставляет функции для работы со временем:
    - Получить текущее время 
    - Перевести наносекунды в секунды
    - Перевести секунды в наносекунды
    - ...
